code(一) HTML 面试核心知识点

HTML (HyperText Markup Language) 是构建 Web 页面的骨架。掌握其核心概念和最佳实践对于前端和全栈开发至关重要。本节梳理了面试中常见且重要的 HTML 基础知识。

💡 提示: 点击左侧导航可以快速跳转。括号内标注了知识点的重要程度。悬停在 关键术语 上可查看解释。尝试章节末尾的测验巩固知识。

bookmarkHTML 语义化 高频 基础

什么是语义化标签？

语义化标签是指那些自身就带有明确含义的 HTML 标签。它们清晰地描述了其内容的角色或结构，例如 <header> 表示页眉，<nav> 表示导航链接，<article> 表示独立的文章内容。

使用语义化标签的主要目的是为了让机器（如搜索引擎、屏幕阅读器）和人（开发者）都能更好地理解网页的结构和内容。

为什么要使用语义化标签？

  • SEO (搜索引擎优化): 搜索引擎更容易理解页面结构，有助于提升排名。
  • 可访问性 (a11y): 屏幕阅读器等辅助技术可以更好地解析页面，为残障人士提供更好的体验。
  • 代码可读性与可维护性: 结构清晰，方便开发者理解和协作。

常用语义化标签及区别

以下是一些 HTML5 中引入的常用语义化标签：

<body>
    <header>... 页眉 (Logo, 标题, 主导航) ...</header>
    <nav>... 主要导航链接 ...</nav>
    <main>... 页面的主要内容 ...
        <article>... 独立的、完整的内容块 (博客文章, 新闻) ...
            <h2>文章标题</h2>
            <section>... 文章内的一个章节 ...</section>
            <section>... 另一个章节 ...</section>
        </article>
        <aside>... 侧边栏 (相关链接, 广告) ...</aside>
    </main>
    <footer>... 页脚 (版权, 联系方式) ...</footer>
</body>

<article> vs <section> 的区别 (重要):

  • <article>: 代表一个独立的、自包含的内容单元。理论上，它可以被单独提取出来发布或重用（如 RSS feed 中的一项）。例如：一篇完整的博客文章、一个论坛帖子、一条用户评论。
  • <section>: 代表文档或应用程序中的一个通用章节或主题性内容分组。它通常包含一个标题 (<h2>-<h6>)。它不像 <article> 那样具有独立的意义，而是构成更大整体的一部分。例如：文章中的“介绍”、“方法”、“结论”部分；网页上的“最新产品”、“联系我们”区域。
  • 判断方法： 问问自己，这块内容单独拿出来是否有意义？如果是，用 <article> 可能更合适。如果只是文档结构的一个部分，用 <section>。它们可以互相嵌套。

面试要点：理解语义化的目的 (SEO, a11y, 可读性) 比单纯记住标签更重要。能清晰解释 <article> 和 <section> 的区别是常见考点。

quiz语义化测验： 在一个博客页面中，包含作者信息、发布日期和相关标签的侧边栏区域，最适合使用哪个语义化标签？

答案: C. <aside>

解析: <aside> 标签用于表示与页面主要内容（博客文章本身）相关，但可以独立存在的内容。作者信息、标签云、相关文章链接等放在侧边栏，正是 <aside> 的典型应用场景。

checklist文档基础与可访问性 高频 基础

<!DOCTYPE html>

作用：位于 HTML 文档最顶部，用于告知浏览器文档类型，触发浏览器的 标准模式 (Standards Mode)。

重要性：必须 声明，否则浏览器可能进入 怪异模式 (Quirks Mode)，导致兼容性问题。

<!DOCTYPE html> <!-- HTML5 的声明非常简洁 -->

面试要点：知道其作用是触发标准模式。

<meta name="viewport">

作用：控制浏览器视口 (viewport) 的大小和缩放，是实现响应式设计的关键。

<meta name="viewport" content="width=device-width, initial-scale=1.0">
  • width=device-width: 视口宽度等于设备宽度。
  • initial-scale=1.0: 初始缩放比例为 1。

面试要点：理解其对响应式布局的基础作用。

Web 可访问性 (a11y) 基础

目标：让网站对所有人可用，包括残障人士和使用辅助技术的用户。

HTML 实践：

  • 使用语义化标签：提供结构和含义。
  • 图像 alt 属性：为 <img> 提供必须的、描述性的替代文本。装饰性图片使用 alt=""。
  • 表单标签 (<label>): 使用 for 属性关联控件 id。
  • 语言声明 (lang): 在 <html> 标签上设置，如 lang="zh-CN"。
  • (进阶) ARIA 属性: 使用 role, aria-* 为复杂组件提供额外语义。

面试要点：理解 a11y 的重要性，知道如何通过 HTML 基础（语义化, alt, label）改进可访问性。了解 ARIA 是加分项。

quiz基础测验： 为了确保网页在移动设备上正确显示并启用响应式布局，通常需要在 <head> 中添加哪个 <meta> 标签？

答案: C. <meta name="viewport" content="width=device-width, initial-scale=1.0">

解析: viewport meta 标签用于控制视口的宽度和初始缩放比例，是移动端网页适配和响应式设计的关键设置。

list_alt表单 (Forms) 高频 基础

表单基础与提交

<form> 用于收集用户输入。关键属性：

  • action: 数据提交的 URL。
  • method: HTTP 方法 (GET 或 POST)。
    • GET: 数据附加到 URL (查询字符串)，用于搜索、非敏感数据，长度受限。
    • POST: 数据在请求体中，更安全，无长度限制，用于提交敏感/大量数据。
  • enctype: 当 method="post" 时的数据编码类型。
    • application/x-www-form-urlencoded (默认): 空格转为 +, 特殊字符编码。
    • multipart/form-data: 文件上传时必须使用，不对字符编码。
    • text/plain: 空格转为 +, 但不对特殊字符编码 (较少使用)。

常用表单控件与可访问性

核心控件包括 <input> (多种 type), <textarea>, <select>, <button>。

可访问性关键：<label>

  • 必须 为每个表单控件提供关联的 <label>。
  • 使用 for 属性指向控件的 id。
  • 好处：点击标签可聚焦控件；屏幕阅读器可以读出标签与控件的关联。
<div class="mb-4">
  <label for="emailInput">邮箱地址:</label>
  <input type="email" id="emailInput" name="email" required placeholder="例如: user@example.com">
</div>

<div class="mb-4">
  <label>
    <input type="checkbox" name="subscribe" value="yes" checked>
    订阅邮件通知
  </label> <!-- 也可以将 input 包裹在 label 内 -->
</div>

面试要点：理解 GET vs POST 区别；知道文件上传需要 method="post" 和 enctype="multipart/form-data"；强调 <label> 对可访问性的重要性。

quiz表单测验： 当表单需要上传文件时，<form> 标签必须设置哪两个属性？

答案: C. method="post", enctype="multipart/form-data"

解析: 文件上传必须使用 POST 方法，因为 GET 方法有长度限制且不适合传输二进制数据。同时，必须将 enctype 设置为 multipart/form-data，这样浏览器才不会对文件内容进行编码，而是将其作为独立部分发送。

image图像 (Images) 高频 基础

<img> 标签核心属性

  • src (必需): 图像路径。
  • alt (必需): 极其重要 的替代文本。
    • 为无法看到图像的用户（如使用屏幕阅读器）提供描述。
    • 图像加载失败时显示。
    • 有助于 SEO。
    • 面试官非常看重 alt 的正确使用。 装饰性图片应使用 alt=""。
  • width / height: 图像尺寸。
    • 推荐设置： 即使使用 CSS 控制，设置这两个属性也能帮助浏览器预留空间，防止CLS。
  • loading="lazy": 浏览器原生懒加载，提升性能。
<img src="product.jpg" alt="红色运动鞋侧面视图" width="400" height="300" loading="lazy">

响应式图像 (srcset & <picture>)

目的：根据设备特性（分辨率、视口宽度、网络状况、格式支持）提供最优图像。

  • srcset: 提供基于分辨率 (x 描述符) 或 视口宽度 (w 描述符 + sizes 属性) 的备选图像源。
  • <picture>: 提供更精细的控制（艺术指导、格式选择），允许根据媒体查询或 type 属性选择不同的 <source>。 
<!-- srcset 示例 (宽度描述符) -->
<img src="cat-small.jpg"
     srcset="cat-small.jpg 480w, cat-medium.jpg 800w, cat-large.jpg 1200w"
     sizes="(max-width: 600px) 90vw, (max-width: 900px) 50vw, 33vw"
     alt="一只可爱的猫">

<!-- picture 示例 (格式选择) -->
<picture>
   <source srcset="logo.avif" type="image/avif"> <!-- 优先 AVIF -->
   <source srcset="logo.webp" type="image/webp"> <!-- 其次 WebP -->
   <img src="logo.png" alt="公司 Logo"> <!-- 最后 PNG -->
</picture>

面试要点：强调 alt 的重要性和正确用法；解释设置 width/height 对防止 CLS 的作用；理解 srcset 和 picture 的基本用途和区别。

quiz图像测验： 对于一张纯粹用于页面装饰、没有实际信息内容的背景图片，<img> 标签的 alt 属性应该如何设置？

答案: C. alt=""

解析: 对于纯装饰性、不传达内容的图片，应提供一个空的 alt 属性 (alt="")。这会告诉屏幕阅读器等辅助技术忽略该图像，避免读出无意义的文件名或造成干扰。省略 alt 属性是不符合规范的，而提供 "装饰图片" 或 "background" 这样的文本则会给用户带来不必要的干扰信息。

link链接与列表 高频 基础

链接 <a>

  • href: 目标 URL。
  • target="_blank": 在新标签页打开。
  • rel="noopener noreferrer": 与 target="_blank" 配合使用以提高安全性。
    • noopener: 防止新页面通过 window.opener 访问来源页面。
    • noreferrer: 防止发送 Referer 头。
  • download: 提示浏览器下载文件。
<a href="https://example.com" target="_blank" rel="noopener noreferrer">安全地在新标签页打开</a>

面试要点：理解 target="_blank" 的安全隐患及 rel="noopener noreferrer" 的作用。

列表 (<ul>, <ol>, <li>)

  • <ul> (Unordered List): 无序列表 (项目符号)。
  • <ol> (Ordered List): 有序列表 (编号)。
  • <li> (List Item): 列表项。
  • <dl>, <dt>, <dd> (Definition List): 定义列表（术语/描述），相对少用但特定场景有用。

列表常用于导航菜单、功能列表、步骤说明等。

quiz链接测验： 当使用 <a target="_blank" href="..."> 打开一个新标签页时，为了防止潜在的安全风险，应该添加哪个 rel 属性值？

答案: B. rel="noopener"

解析: rel="noopener" 可以阻止新打开的页面通过 window.opener 属性访问和操作原始页面，从而防止一种称为 "tabnabbing" 的钓鱼攻击。通常会同时添加 noreferrer (rel="noopener noreferrer") 来阻止发送 Referer 头，进一步增强隐私和安全。

table_chart表格 (Tables) 中频 基础

表格结构与语义

用于展示二维数据。正确的结构有助于可访问性和样式化。

  • 结构标签: <table>, <caption>, <thead>, <tbody>, <tfoot>, <tr>, <th>, <td>.
  • <th> 的 scope 属性: 重要，用于明确表头是作用于行 (row) 还是列 (col)，增强可访问性。
  • colspan / rowspan: 用于合并单元格。

面试要点：理解表格的语义化结构；知道 scope 属性的作用；强调表格不应用于布局。

space_bar块级/内联/盒模型 高频 基础

块级 (Block) vs 内联 (Inline)

特性 块级 (Block) 内联 (Inline)
换行 独占一行 同行显示
宽高 可设置 width/height 通常不可设置 width/height
外边距 上下左右 margin 有效 左右 margin 有效，上下通常无效
内边距 上下左右 padding 有效 上下 padding 可能影响行高，左右有效
示例 <div>, <p>, <h1>, <ul>, <section> <span>, <a>, <strong>, <em>, <img>*
* <img> 等可替换元素表现类似 inline-block

内联块 (Inline-block): 通过 display: inline-block; 设置，结合两者特性：同行显示，但可设置宽高内外边距。

CSS 盒模型 (Box Model)

虽然是 CSS 概念，但与 HTML 元素布局紧密相关。

每个 HTML 元素都可以看作一个矩形盒子，由内到外包含：

  • Content: 元素的实际内容（文本、图像等）。
  • Padding: 内容区域与边框之间的空白区域（内边距）。
  • Border: 包围内边距和内容的边框。
  • Margin: 边框与其他元素之间的空白区域（外边距）。

box-sizing 属性 (重要):

  • content-box (默认): width 和 height 只包含 Content 的尺寸。Padding 和 Border 会额外增加元素的总尺寸。
  • border-box: width 和 height 包含 Content、Padding 和 Border 的总尺寸。这是现代 Web 开发中更常用、更直观的设置，通常在全局设置：
/* 通常放在 CSS 重置或全局样式中 */
*,
*::before,
*::after {
  box-sizing: border-box;
}

面试要点：清晰描述块级/内联区别；理解标准盒模型和 border-box 盒模型的区别及其影响，知道 box-sizing: border-box; 的作用和常用实践。

quiz盒模型测验： 如果一个元素的 CSS 设置为 width: 100px; padding: 10px; border: 1px solid black; 并且 box-sizing: border-box;，那么该元素在页面上占据的总宽度是多少？

答案: A. 100px

解析: 当 box-sizing 设置为 border-box 时，width 属性定义的宽度包含了内容（content）、内边距（padding）和边框（border）。因此，即使有 padding 和 border，元素占据的总宽度仍然是 width 属性指定的值，即 100px。内边距和边框会向内挤压内容区域的空间。

extension其他重要特性 中频 基础

<iframe> 内联框架

用于在当前页面嵌入另一个 HTML 页面。

  • 优点: 可以嵌入第三方内容、实现无刷新更新部分页面（旧技术）。
  • 缺点:
    • 阻塞父页面的 onload 事件。
    • 可能带来安全风险（需要注意跨域限制）。
    • SEO 不友好，搜索引擎可能难以索引 iframe 内容。
    • 性能开销，每个 iframe 都相当于一个独立的浏览上下文。
  • sandbox 属性 (重要): 用于限制 iframe 的行为，提高安全性。可以设置允许的操作，如 allow-scripts, allow-same-origin, allow-forms 等。如果只设置 sandbox 而不指定值，则禁用所有权限。 
<!-- 嵌入一个页面，并限制其行为 -->
<iframe src="https://example.com"
        sandbox="allow-scripts allow-same-origin"
        width="600"
        height="400"
        title="嵌入的示例页面">
</iframe>

面试要点：了解 iframe 的基本用途、主要缺点（阻塞 onload、安全、SEO、性能）以及 sandbox 属性的作用。

data-* 自定义数据属性

允许在 HTML 元素上存储自定义数据，这些数据可以被 CSS 或 JavaScript 访问。

  • 属性名必须以 data- 开头，后面至少跟一个字符。
  • 属性名不应包含大写字母，单词间用短横线 (-) 分隔。
  • 属性值可以是任何字符串。

访问方式：

  • CSS: 使用 attr() 函数，如 content: attr(data-tooltip);。
  • JavaScript: 通过元素的 dataset 属性访问。属性名会自动转换为驼峰式命名 (camelCase)。 
<div id="user-card" data-user-id="123" data-user-role="admin">
  用户卡片
</div>

<script>
  const card = document.getElementById('user-card');
  console.log(card.dataset.userId); // 输出: "123"
  console.log(card.dataset.userRole); // 输出: "admin"

  // 修改
  card.dataset.userRole = 'editor';
  console.log(card.getAttribute('data-user-role')); // 输出: "editor"
</script>

面试要点：知道 data-* 属性的用途（存储自定义数据）和如何在 JavaScript 中通过 dataset 访问（注意命名转换）。

通用容器 (<div> & <span>)

<div> (块级) 和 <span> (内联) 是无具体语义的通用容器，主要用于：

  • 布局: 配合 CSS (Flexbox, Grid) 构建页面结构。
  • 样式化: 包裹元素以应用特定 CSS 样式。
  • JavaScript 操作: 作为通过 ID 或 Class 选择和操作的目标。

注意：应优先使用语义化标签。只有在没有更合适的语义化标签时，才使用 <div> 或 <span>。

quiz其他特性测验： 在 JavaScript 中，如何访问 HTML 元素 <article data-post-id="987">...</article> 的 data-post-id 属性值？

答案: B. element.dataset.postId

解析: HTML5 提供了 dataset 属性来方便地访问元素的 data-* 自定义属性。访问时，data- 后面的属性名会从短横线命名 (kebab-case) 转换为驼峰式命名 (camelCase)。因此，data-post-id 对应 element.dataset.postId。
